# AGENTS.md — SCIAM Tech Blog

Jekyll 4.3 static blog for [blog.sciam.fr](https://blog.sciam.fr).  
Primary post format: **AsciiDoc** (`.adoc`). Markdown (`.md`) is supported but secondary.  
CSS is compiled separately with Tailwind — it is **not** rebuilt automatically by Jekyll.

---

## Developer Workflows

```sh
# Start dev server (installs all deps, serves drafts, livereload on :4321)
./runServer.sh

# Rebuild Tailwind CSS (required after changing layouts/includes/HTML classes)
./gentailwind.sh

# Clean Jekyll cache
./clean.sh
```

Dev server flags used: `--incremental --livereload --drafts --open-url`  
Production build: `bundle exec jekyll build`

## Creating a Post

### AsciiDoc (preferred)

File: `_posts/YYYY-MM-DD-slug.adoc`

```asciidoc
= Post Title
:showtitle:
:page-navtitle: Post Title
:page-excerpt: One-sentence description shown in listing cards.
:layout: post
:author: authorkey
:page-tags: [Tag1, Tag2]
:page-vignette: filename.png
:page-vignette-licence: 'Source ...'
:page-categories: software
:page-liquid:

Content starts here.

//more

Rest of post after excerpt cut.
```

- Excerpt separator: `//more` (on its own line, surrounded by blank lines)
- Multi-author: use `:page-auteurs: [key1,key2]` instead of `:author:`
- Disable TOC: `:page-toc: false` (TOC is on by default for `_posts`)
- Video posts: add `:page-categories: video` and `:vimeo-id: <id>`

### Markdown (secondary)

File: `_posts/YYYY-MM-DD-slug.md`

```yaml
---
layout: post
title: "Post Title"
author: authorkey
tags: ['Tag1', 'Tag2']
excerpt: "One-sentence description."
vignette: filename.png
categories: software
toc: true
---
```

Excerpt separator: `<!--more-->`

## Authors

Author keys are defined in `_data/authors.yml` as lowercase concatenated names  
(e.g., `antoinesabotdurand`, `jeanmicheldoudoux`).  
Required fields: `name`, `picture`; optional: `bio`, `job`, `pagesciam`, `socials`.  
Author pages are auto-generated by `jekyll-auto-authors` at `/author/<key>/`.

## Categories & Tags

Valid categories (from `_data/navigation.yml` → `_config.yml` autopages):

| Key           | Nav label            |
|---------------|----------------------|
| `software`    | Software Engineering |
| `llm`         | Data, IA & LLM       |
| `sciencesco`  | Sciences co & Cyber  |
| `autres`      | Autres               |
| `video`       | Vidéos               |

A post can belong to multiple categories: `:page-categories: software llm`  
Tag pages are auto-generated at `/tag/<slug>/`.

## CSS & Tailwind

- Source: `css/tailwind-source.css` → compiled to `css/styles.css`
- Run `./gentailwind.sh` after editing any HTML template (layouts/includes)
- Tailwind scans only `_layouts/**/*.html`, `_includes/**/*.html`, `*.html`
- Custom brand colors: `navysciam`, `navysciamdark`, `navysciamlight`, `yellowsciam`, `lightbluesciam`
- Custom breakpoints: `nav` (1150px), `md` (820px), `tablet` (820–1024px)
- Custom fonts: `font-sans` → Noyh Geometric; `font-asciidoc` → Droid Sans Mono

## Project Structure

```
_posts/          # Published posts (YYYY-MM-DD-slug.adoc|md)
_drafts/         # Unpublished drafts (no date prefix required)
_layouts/        # post.html → default.html; list.html for index/tag/author pages
_includes/       # Reusable partials (postbox.html, author_bio.html, …)
_data/
  authors.yml    # Author registry (keyed by lowercase-concatenated name)
  navigation.yml # Top nav items + valid categories
images/vignettes/  # Post thumbnail images referenced by :page-vignette:
css/styles.css   # Compiled Tailwind output — do not edit directly
```

## Key Plugins

| Plugin                  | Role                                              |
|-------------------------|---------------------------------------------------|
| `jekyll-asciidoc`       | AsciiDoc rendering with Asciidoctor               |
| `asciidoctor-diagram`   | PlantUML/Mermaid diagrams in AsciiDoc posts       |
| `jekyll-paginate-v2`    | Pagination (12 posts/page) + auto category pages  |
| `jekyll-auto-authors`   | Author pages from `_data/authors.yml`             |
| `jekyll-toc`            | Sidebar table of contents via `{{ content | toc_only }}` |
| `lunr`                  | Client-side full-text search                      |
| `liquid_reading_time`   | `{{ content | reading_time }}` in post header     |

## Notes

- Comments use **Giscus** (GitHub Discussions) — no server-side component needed.
- Analytics is included only when `jekyll.environment == 'production'`.
- `_site/` is the build output — do not edit files there directly.
- The `asciidoctor` config sets `safe: unsafe` to allow includes and diagrams.
